このブログを作った話(計画編)
こんにちは、このブログを運営(?)している、RasPython3です。最近はらすぴーとも呼んでもらっています。

さて、おとといのことになりますが、ふと思い立ち、「ブログ作っちゃおう」となりました。
意味不明です。ちなみに試験期間中です。頭おかしいです。どちらかというと頭がおかしくなった結果です。

...今ですか？もちろん試験期間中ですよ？

で、思い立ったはいいものの、サーバーを借りたりするとお金がかかるので、サーバーは借りなかったです。
じゃどーすんの？既存のブログサイト使うの？となりますが、今回は「作る」ことが目的なので却下です。
さて。残る方法は、知っているものでGASかGitHub Pagesぐらいですが。

GASは、googleがヘッダー埋め込んできたり制限多かったりで不便です。
一方、GitHub Pagesは静的サイトのみです。

ここで思ったんですけど。
ブログって返信機能とか作らなければ静的サイトでもどうにかなるじゃないですか。
どうにかなりますよね？ね？(圧)

と、いうわけで。採用されたのは、GitHub Pagesでした。

「いや、投稿機能作れないじゃん、無理でしょ」って思った方。そこのあなたです。たぶん。

逆に考えてみましょう。
「**稿**」を「**投**」げる必要が、どこにあるんでしょうか。
ないですよね。

じゃあどうやってブログを書くのか？
**だってgitがあるじゃない。GitHubなんだから。**

「は？」という方向けにかんたんに説明しますと、
```
1. 記事を書く。
2. スクリプトでhtmlに変換する。
3. リポジトリにgit pushする。
4. GitHub Pagesで見れるようになる。
```
というわけです。かんたんですね。
